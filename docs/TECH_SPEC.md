# Техническая спецификация: Image Improver

## 1. Описание системы

Система улучшения изображений в реальном времени, работающая в браузере пользователя. ML-модель анализирует изображение и подбирает оптимальные параметры коррекции (яркость, контрастность, цветность), вспомогательный алгоритм применяет эти параметры через GPU.

## 2. Функциональные требования

| № | Требование | Описание |
|---|-----------|----------|
| 1 | Загрузка изображения | Пользователь загружает файл с устройства (File / Blob) |
| 2 | Поддержка форматов | JPG, PNG, HEIC, BMP |
| 3 | ML-анализ | Модель предсказывает 3 параметра: яркость, контраст, цветность |
| 4 | Применение параметров | WebGL2-шейдер применяет параметры к изображению |
| 5 | Выгрузка результата | JPEG по умолчанию, PNG при наличии прозрачности |
| 6 | Прогресс обработки | События API информируют о статусе и прогрессе |
| 7 | Прерывание задачи | Возможность отменить обработку на любом этапе |

## 3. Нефункциональные требования

| № | Требование | Обоснование |
|---|-----------|-------------|
| 1 | Суммарный объём кода (бандл) не должен превышать 10 МБ | Ограничение ТЗ для обеспечения быстрой загрузки |
| 2 | Максимальное время обработки одного изображения не должно превышать 30 секунд | Ограничение ТЗ для приемлемого пользовательского опыта |
| 3 | Среднее время обработки изображения не должно превышать 5 секунд | Ограничение ТЗ для интерактивной работы |
| 4 | Система должна поддерживать обработку изображений разрешением до 15 мегапикселей | Ограничение ТЗ для работы с фотографиями высокого качества |
| 5 | Система должна работать на среднем ноутбуке (процессор Intel i5 / AMD Ryzen 5, 8 ГБ оперативной памяти, интегрированная графика) | Ориентация на худший случай для гарантии работоспособности у среднестатистического пользователя |
| 6 | Система должна быть работоспособна во всех массовых современных браузерах: Chrome, Firefox, Safari, Edge | Ограничение ТЗ для максимального охвата пользователей |

## 4. Технологический стек

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Сборка | Vite + TypeScript | Быстрая сборка, строгая типизация |
| ML-движок | ONNX Runtime Web (WASM) | Лёгкий, поддержка SIMD, квантование INT8 |
| ML-модель | MobileNetV3-Small / MLP | Сверхлёгкая архитектура, ~2 МБ в INT8 |
| Обработка изображений | WebGL2 (фоллбэк WebGL) | GPU-ускорение, миллисекунды на 15 Мп |
| Декодинг HEIC | libheif WASM (lazy load) | Динамическая загрузка — не раздувает основной бандл |
| Декодинг JPG/PNG/BMP | WebCodecs API / нативный | Быстрое нативное декодирование |
| Асинхронность | Web Worker + OffscreenCanvas | Без блокировки UI |
| Передача данных | Transferable Objects | Предотвращение копирования 60 МБ буферов |

## 5. Стратегия кодирования результата

| Исходный формат | Прозрачность | Результат |
|-----------------|--------------|-----------|
| JPG | — | JPEG (92%) |
| PNG | Нет | JPEG (92%) |
| PNG | Есть | PNG |
| HEIC | — | JPEG (92%) |
| BMP | — | JPEG (92%) |

Пользователь может явно запросить формат через параметр `format: 'jpeg' | 'png'`.

## 6. Ограничения и допущения

1. **Формат результата не сохраняется.** HEIC и BMP конвертируются в JPEG/PNG для обеспечения практической работоспособности.
2. **Качество сжатия.** JPEG с качеством 92% обеспечивает визуально неотличимый результат от оригинала для обычного пользователя (потери заметны только при профессиональном анализе на уровне пикселей).
3. **WebGPU не используется.** Ориентируемся на WebGL2 для совместимости со средним ноутбуком.
4. **ML-модель предобученная.** Дообучается на MIT-Adobe FiveK (около 2800 изображений).

## 7. Безопасность и приватность

1. **Локальная обработка.** Все изображения обрабатываются исключительно в браузере пользователя. Данные не передаются на внешние серверы.
2. **Отсутствие сетевых запросов.** После загрузки ML-модели и WASM-модулей система работает полностью офлайн.
3. **Конфиденциальность.** Пользователь сохраняет полный контроль над своими данными — изображения не покидают устройство.
4. **Безопасность CORS.** Загрузка файлов с устройства через `<input type="file">` исключает проблемы с кросс-доменными запросами.

## 8. Критерии приёмки

| Критерий | Оценка |
|----------|---------|
| Качество изображения | Визуальная оценка + PSNR/SSIM на эталонных изображениях |
| Скорость | Среднее ≤ 5 с, макс ≤ 30 с на среднем ноутбуке |
| RAM | Нет утечек при многократной обработке |
| CPU | Основной поток загружен < 5% |
| Размер бандла | ≤ 10 МБ (с учётом всех WASM-модулей) |
| Кроссбраузерность | Работает в Chrome, Firefox, Safari, Edge |
